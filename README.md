# 自動點擊工具 (Auto Clicker)

一個功能完善的 Python 自動滑鼠點擊工具,支援 GUI 介面控制、座標擷取、統計資訊顯示等功能,適用於遊戲輔助、自動化測試等場景。

## 功能特色

- ✅ **圖形化介面** - 使用 tkinter 建立的友善 GUI 介面
- ✅ **座標擷取** - 點擊按鈕後可直接在螢幕上選取目標位置
- ✅ **可調整間隔** - 支援設定點擊間隔時間 (0.01-10 秒)
- ✅ **統計資訊** - 即時顯示已點擊次數與運行時間
- ✅ **全域熱鍵** - 按 Cmd+Shift+Esc (macOS) 或 Ctrl+Shift+Esc (Windows/Linux) 緊急停止
- ✅ **視窗置頂** - 視窗始終保持在最上層,方便操作
- ✅ **設定管理** - 可儲存/載入常用的座標和參數設定
- ✨ **多桌面支援 (macOS)** - 切換桌面時跳過點擊,滑鼠完全不受干擾
- 🆕 **自動停止** - 可設定點擊次數上限,達到後自動停止
- 🆕 **緊急停止** - 按 Cmd+Shift+Esc (或 Ctrl+Shift+Esc) 立即停止,或點擊醒目的紅色「緊急停止」按鈕

## 系統需求

- **作業系統**: macOS / Windows / Linux
- **Python 版本**: Python 3.7 或以上
- **依賴套件**:
  - `pyautogui` - 滑鼠控制
  - `keyboard` - 全域熱鍵監聽

## 安裝步驟

### 1. 下載專案

將專案檔案下載到本機,或使用 Git clone:

```bash
cd ~/Desktop
# 專案已經在 auto_clicker 資料夾中
```

### 2. 安裝依賴套件

```bash
pip3 install pyautogui keyboard
```

**注意**: 在 macOS 上,`keyboard` 套件需要輔助功能權限,請在「系統偏好設定 > 安全性與隱私權 > 輔助使用」中允許終端機或 Python 存取。

### 3. 執行程式

```bash
cd ~/Desktop/auto_clicker
python3 auto_clicker.py
```

或直接雙擊 `auto_clicker.py` 檔案執行。

## 使用說明

### 基本操作流程

1. **設定座標**
   - 方法一:直接在「X 座標」和「Y 座標」欄位輸入數值
   - 方法二:點擊「選取座標」按鈕,然後在螢幕上點擊目標位置

2. **設定點擊間隔**
   - 在「點擊間隔」欄位輸入秒數 (例如 0.5 表示每 0.5 秒點擊一次)

3. **設定點擊上限** (選填)
   - 在「點擊上限」欄位輸入想要的點擊次數
   - 設定為 0 表示無限點擊

4. **開始點擊**
   - 點擊「開始點擊」按鈕,程式將開始在指定座標自動點擊
   - 統計資訊會即時更新顯示點擊次數和運行時間
   - 如果設定了點擊上限,會顯示進度比例（例如：234/1,000 次）
   - 終端機會每 5 秒輸出一次進度資訊

5. **停止點擊**
   - 點擊「⚠️ 緊急停止」按鈕
   - 或按 Cmd+Shift+Esc (macOS) / Ctrl+Shift+Esc (Windows/Linux) 緊急停止

6. **儲存/載入設定**
   - 點擊「儲存設定」將當前參數存入 `config.json`
   - 點擊「載入設定」從 `config.json` 讀取先前儲存的設定
   - 程式啟動時會自動載入上次的設定

### 快捷鍵

| 熱鍵  | 功能           | 說明                                     |
| ----- | -------------- | ---------------------------------------- |
| **Cmd+Shift+Esc** (macOS)<br>**Ctrl+Shift+Esc** (Win/Linux) | 緊急停止    | 立即完全停止點擊,重置程式狀態 |

**注意**:
- 全域熱鍵在任何視窗下都有效,即使滑鼠正在點擊目標位置也能使用

- 也可以點擊 GUI 上的紅色「⚠️ 緊急停止」按鈕

### 🖥️ macOS 多桌面功能說明

在 macOS 上,如果您使用 Mission Control 的多個桌面 (Spaces) 功能,本工具提供了智慧的桌面檢測機制:

#### 功能特性

1. **視窗固定在單一桌面**
   - 程式視窗只會出現在您開啟程式時的桌面
   - 不會跟隨到其他桌面,避免干擾其他工作空間

2. **智慧桌面檢測**
   - 使用 macOS CGWindowListCopyWindowInfo API 檢測視窗是否在當前桌面
   - 當您切換到其他桌面時,**自動跳過點擊動作**
   - **滑鼠不會被移動或點擊**,完全不干擾其他桌面的操作
   - 在同一桌面內,即使點擊其他應用（如瀏覽器）,點擊仍會繼續

3. **無縫工作流程**
   - 您可以在同一桌面同時使用瀏覽器和自動點擊工具
   - 切換到其他桌面時,滑鼠不會被干擾
   - 完全自動化,不需要手動暫停/恢復

#### 使用範例

```
桌面 1: 開啟瀏覽器執行網頁遊戲
        執行自動點擊工具,在瀏覽器視窗上點擊
        → 正常點擊中,即使點擊瀏覽器使用遊戲,點擊仍會繼續

切換到桌面 2:
        → 程式視窗不在當前桌面
        → 自動跳過點擊,滑鼠完全不受影響

桌面 2: 處理工作或瀏覽網頁
        → 滑鼠可以自由使用,不會被移動或點擊

切換回桌面 1:
        → 程式視窗回到當前桌面
        → 自動恢復點擊
```

#### 技術實現

- 使用 CGWindowListCopyWindowInfo API 檢測視窗是否在螢幕上可見
- 當視窗不在當前桌面時,kCGWindowIsOnscreen 為 False
- 準確區分「同桌面失去焦點」和「切換到其他桌面」

#### 注意事項

- 此功能僅在 macOS 上啟用
- Windows/Linux 系統使用傳統的全域置頂模式
- 需要安裝 `pyobjc-framework-Cocoa` (已自動包含在依賴中)

## 設定檔格式

程式會在同目錄下建立 `config.json` 檔案,格式如下:

```json
{
    "x": 100,
    "y": 200,
    "interval": 0.5,
    "max_clicks": 100,
    "last_updated": "2025-11-12 12:30:45"
}
```

**參數說明**:
- `x`, `y`: 點擊座標
- `interval`: 點擊間隔(秒)
- `max_clicks`: 點擊上限次數 (0=無限) 🆕
- `last_updated`: 最後更新時間

## 技術架構

### 核心模組

1. **ConfigManager** - 設定檔管理
   - 儲存/載入 JSON 格式的設定檔

2. **StatisticsTracker** - 統計追蹤
   - 計數點擊次數
   - 計算運行時間

3. **ClickController** - 點擊控制
   - 管理自動點擊執行緒
   - 處理暫停/恢復邏輯
   - 整合桌面狀態檢查

4. **CoordinateCapture** - 座標擷取
   - 捕捉使用者點擊位置

5. **DesktopMonitor** - 桌面監控 (macOS)
   - 偵測程式所在桌面
   - 判斷是否在當前活躍桌面

6. **AutoClickerGUI** - 圖形介面
   - 整合所有模組
   - 處理使用者互動

## 常見問題

### Q1: macOS 上提示「需要輔助功能權限」怎麼辦?

**解答**:
1. 開啟「系統偏好設定」
2. 進入「安全性與隱私權」
3. 選擇「輔助使用」標籤
4. 解鎖並勾選「終端機」或「Python」
5. 重新執行程式

### Q2: 熱鍵沒有反應?

**解答**:
- 確認 `keyboard` 套件已正確安裝
- 在 macOS 上需要授予輔助功能權限
- 某些遊戲可能會攔截全域熱鍵,請嘗試切換到桌面再按熱鍵
- 也可以直接點擊紅色「⚠️ 緊急停止」按鈕

### Q3: 點擊座標不準確?

**解答**:
- 使用「選取座標」功能可以更精確地選取位置
- 確認顯示器解析度設定正確
- 在多螢幕環境下,座標可能會受到影響

### Q4: 如何停止程式?

**解答**:
- 點擊「⚠️ 緊急停止」按鈕立即停止（推薦）
- 按 Cmd+Shift+Esc (macOS) 或 Ctrl+Shift+Esc (Windows/Linux) 緊急停止
- 關閉視窗即可退出程式

### Q5: 可以同時點擊多個位置嗎?

**解答**: 當前版本僅支援單一固定位置點擊。如需多點輪流點擊功能,請聯繫開發者或自行修改程式碼。

### Q6: macOS 多桌面功能如何運作?

**解答**:
- 程式會自動偵測是否在 macOS 上運行
- 視窗只會固定在開啟時的桌面,不會出現在其他桌面
- **使用 CGWindowListCopyWindowInfo API 檢測視窗是否在當前桌面**
- **切換到其他桌面時,自動跳過點擊動作,滑鼠不會被移動**
- 在同一桌面內,即使點擊其他應用（如瀏覽器）,點擊仍會繼續
- 這個功能完全自動化,不需要手動設定

### Q7: 在桌面 1 點擊瀏覽器,點擊會停止嗎?

**解答**: **不會!** 只要瀏覽器和自動點擊工具在同一個桌面,點擊就會繼續:
- 在同一桌面內點擊其他應用（瀏覽器、遊戲等）→ **點擊繼續**
- 切換到其他桌面 → **跳過點擊,滑鼠不被干擾**
- 程式可以準確區分「同桌面失去焦點」和「切換到其他桌面」
- 這樣您可以在桌面 1 同時使用瀏覽器和自動點擊工具

### Q8: 滑鼠一直在點擊目標位置,無法停止怎麼辦?

**解答**: 現在有多種方式可以停止點擊:
1. **點擊紅色「⚠️ 緊急停止」按鈕** - GUI 上的紅色按鈕（黑色文字），最可靠（推薦）
2. **按 Cmd+Shift+Esc (macOS) 或 Ctrl+Shift+Esc (Windows/Linux)** - 快速的緊急停止
3. **設定點擊上限** - 在「點擊上限」欄位輸入次數,達到後自動停止
4. **關閉視窗** - 直接關閉程式視窗

### Q9: 如何設定自動停止?點擊上限怎麼用? 🆕

**解答**:
- 在「點擊上限」欄位輸入想要的點擊次數(例如 100)
- 設定為 0 表示無限點擊,不會自動停止
- 達到設定的次數後,程式會自動停止並顯示提示訊息
- 適合用於需要固定次數點擊的場景

### Q10: 緊急停止熱鍵按了沒反應?

**解答**:
1. **檢查輔助功能權限** (最常見原因)
   - 在 macOS 上需要授予「輔助使用」權限
   - 前往「系統偏好設定 > 安全性與隱私權 > 輔助使用」
   - 勾選「終端機」或「Python」
   - 重新啟動程式

2. **熱鍵被其他應用攔截**
   - 某些全螢幕應用可能會攔截組合鍵
   - Windows 系統: Ctrl+Shift+Esc 會同時開啟任務管理員
   - 解決方法：
     - 先按 Cmd+Tab (macOS) 或 Alt+Tab (Windows) 切換到桌面
     - 然後按緊急停止熱鍵
     - 或直接點擊紅色「⚠️ 緊急停止」按鈕（最可靠）

3. **使用備用方法**
   - 點擊 GUI 上的紅色「⚠️ 緊急停止」按鈕（最可靠）
   - 直接關閉程式視窗
